Taller de emuladores

¡Gracias por anotarte en el taller!
Antes de venir estaría bueno tener andando un ejemplo básico de aplicaciones gráficas en el lenguaje que elegiste, que por lo menos incluya:

  • Crear una ventana
  • Manejar eventos (detectar cuando se presionan y sueltan las teclas, mouse no hace falta)
  • Dibujar píxeles en pantalla (tener una función setPixel y si se puede getPixel también)

Acá hay algunos recursos que podés ir investigando para realizarlo: (Cualquier duda escribime por Telegram)

C/C++

  • SDL2: voy a usar esta biblioteca para el taller, puedo enviar un ejemplo básico
  • Si preferís compilarlo old-school para x86 de 16 bits con TurboC, podés usar el modo 13h y escribir directo en la memoria de video (0xA0000) y leer del teclado usando el BIOS keyboard buffer
  • Alternativa: Immediate2d: una biblioteca de gráficos tiny, solo Windows
  • Alternativa: Minifb: una biblioteca tiny pero multiplatforma

VB6

Casi que está todo resuelto acá, se pueden usar los eventos KeyDown/KeyUp. Para leer y dibujar píxels se puede agregar un Picturebox en un form.

Rust

  • Recomendado el binding de SDL2 (voy a usar la versión de C de esa biblioteca para el taller)
  • Alternativa: Rust_Minifb, una biblioteca más liviana que SDL

Python

Pygame es la biblioteca típica para este estilo de aplicaciones, soporta set_at en una superficie o un PixelArray para manipular píxeles

MicroPython

Para leer desde las teclas se puede usar distintos pines de GPIO. Como pantalla se puede usar un mini OLED SSD1306, que está soportado nativamente vía I2C o SPI.

C# u otro de .NET

Se puede agregar un componente WriteableBitmap a un formulario, y luego con wBitmap.WritePixels(...) se pueden manipular sus píxeles. Los eventos de teclado se pueden capturar directo en el form.

JavaScript

Lo más sencillo es usar un canvas y manipular los pixeles de la imagen con un ImageData. Los eventos se pueden capturar con onkeydown/onkeyup
